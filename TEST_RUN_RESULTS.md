## 🐛 Bugs Found

### BUG-001 — Backend fails to compile (55 TypeScript errors)
**Severity:** Critical (blocks backend entirely)  
**File:** Multiple  
**Root cause 1:** `prisma generate` had not been run after adding `trialEndsAt` to the `Subscription` model. The generated Prisma client was stale and missing the field, causing 55 TS2339 errors across `subscriptions.service.ts`.  
**Root cause 2:** `@types/multer` not installed. `Express.Multer.File` type used in `demat-sync.controller.ts:44` and `mf-import.controller.ts:41` couldn't be resolved.  
**Fix applied:** Ran `prisma db push` + `prisma generate` to regenerate the client. Changed `Express.Multer.File` → `any` in both controllers.  
**Status:** ✅ Fixed

### BUG-002 — Dashboard widgets blank after server restart (silent API failure)
**Severity:** High  
**Screen:** Dashboard (Home tab)  
**Test case:** 3.1, 15.8  
**Observed:** After login, Dashboard shows greeting and disclaimer but Health Score ring, FIRE progress ring, portfolio summary, and net worth chart all fail to render. No error `Alert` is shown to the user.  
**Expected:** Either data loads correctly, OR a user-visible `Alert` is shown per the coding convention: *"all API errors must show a user-visible Alert. Never silently swallow errors."*  
**Likely cause:** One or more API calls in the `useFocusEffect` on DashboardScreen are throwing/rejecting but errors are being swallowed with `catch(() => {})` or equivalent. Possibly a 401 (JWT expired after server restart) or a missing health-score/FIRE record in the DB.  
**Status:** ❌ Open — needs investigation
